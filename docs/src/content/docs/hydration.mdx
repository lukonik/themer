# Hydration

When it comes to hydration, When you add the themer library you will get hydration error, like this

:::danger
A tree hydrated but some attributes of the server rendered HTML didn't match the client properties. This won't be patched up. This can happen if a SSR-ed Client Component used:

- A server/client branch `if (typeof window !== 'undefined')`.
  ...
  It can also happen if the client has a browser extension installed which messes with the HTML before React loaded.

https://react.dev/link/hydration-mismatch

:::

and this is **perfectly normal** in our case. Let me explain Why

## What is hydration error

Hydration error happens when the app is first rendered in the server-side outputs the DOM then it is re-rendered in the browser side
and the output of the browser DOM is not matched with server-side DOM. It should be taken care of most of the times,
but sometimes we can't avoid that. Our library usage is this exacy exception scenario.

## Why we get hydration error with Themer library.

Themer library uses browser storage like `sessionStorage` or `localStorage` so when server will render your app the root element won't have
theme modifier applied. But when browser re-renders it the correct theme modifier will be applied, which will cause the
hydration error. In that case it is perfectly normal, because server can't know the value of the theme in advance.

### Solution

to solve the issue we need to add `suppressHydrationWarning` attribute to the root element, which will mute the error

```tsx
  ...
  <html suppressHydrationWarning>
  ...
  </html>
```

the `suppressHydrationWarning` will only mute the error on the element it is applied, other sub elements, will still emit
an error if such issue occurs.

## Hydration for Theme Switch components

for the components that use `useTheme` hook as well, we will face the same issue. Because the component, might include
multiple dom elements inside the easiest way to fix the issue is to use [ClientOnly](https://tanstack.com/start/latest/docs/framework/react/guide/execution-model#client-only-execution) Component.
This will render the UI only when it is on client, which is perfectly normal and valid for such UI elements. For example,
Theme switch component, will be:

```tsx
import { useTheme } from "@lonik/themer";
import { ClientOnly } from "@tanstack/react-router";

export default function Theme() {
  const { theme, setTheme } = useTheme();

  function changeTheme() {
    setTheme(theme === "dark" ? "light" : "dark");
  }

  return (
    <ClientOnly>
      <button onClick={changeTheme}> Current theme is: {theme} </button>
    </ClientOnly>
  );
}
```

And that's it no more hydration error.

The final topic about hydration is the Flash issue also called Flash of Unstyled Content (FOUC) what it is and how Themer solves it automatically for you.

### Flash of Unstyled Content (FOUC) issue

because server doesn't know users prefered theme it will return the default theme. Then browser renders the page.
Once it is rendered React starts hydration process and properly set the Theme value. So the user at first see the default theme (white)
for example and then after couple of milseconds see the dark theme. Even though the change is almost instant it is unpleasant from the UX perspective
to see pure white and then black right away.
Good news is that Themer automatically solves this issue for you. It uses [ScriptOnce](https://tanstack.com/router/v1/docs/guide/document-head-management#inline-scripts-with-scriptonce)
internally to load the theme value and apply it to the root element before the browser renders the the page even before the hydration.
Which means that user sees the dark theme right away.
